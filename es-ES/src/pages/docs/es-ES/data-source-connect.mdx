Aprende sobre cómo conectarte a tu fuente de datos y llámalo en tu Constructor de ILLA.

ILLA le permite conectar e integrar una amplia gama de fuentes de datos, incluyendo bases de datos y APIs, en su organización para su uso como recursos.

## Fuentes de datos

Las fuentes de datos son lugares donde se almacenan y procesan los datos. Ejemplos de fuentes de datos incluyen bases de datos PostgreSQL, API Twilio REST y cubos Amazon S3. En ILLA puede agregar múltiples fuentes de datos y hacerlas accesibles a los usuarios para consultar.

### **Integraciones**

ILLA proporciona integraciones integradas para muchas fuentes de datos populares que requieren una mínima configuración, como bases de datos SQL y Stripe. Estas integraciones, también conocidas como recursos, le permiten conectarse y consultar sus fuentes de datos usando Retool. Si un tipo particular de fuente de datos no es soportado por una integración integrada, puede usar las integraciones REST, GraphQL o SOAP para conectarse a casi cualquier API. Esto le permite acceder a una amplia gama de fuentes de datos desde dentro del constructor ILLA.

## Recursos

Un recurso en ILLA es una configuración guardada que especifica cómo se conecta ILLA a una fuente de datos. Para utilizar una fuente de datos en ILA, debe crear un recurso para ella. Puede seleccionar el recurso apropiado para acceder a los datos al escribir una consulta.

Cada recurso de ILLA tiene sus propias opciones de configuración que determinan cómo se conecta ILLA a la fuente de datos correspondiente. Estas opciones pueden incluir credenciales y flujos de autenticación, configuración de conexión de base de datos y configuración de API, como URL base y cabeceras predeterminadas.

### **Cómo funcionan los recursos**

Los recursos de ILLA almacenan los ajustes necesarios para conectarse a su fuente de datos de forma segura. Esto le permite consultar datos fácilmente sin exponer su fuente de datos directamente a sus usuarios. Cuando se ejecuta una consulta para un recurso, Retool utiliza los ajustes de configuración del recurso para proxy de la petición a la fuente de datos del lado del servidor. Esto asegura que sólo Retool tiene acceso directo a la fuente de datos.

ILLA simplifica el proceso de conexión a una fuente de datos para sus usuarios. Sólo necesitan seleccionar el recurso relevante al escribir una consulta sin preocuparse por los detalles de la fuente de datos. También puede agregar múltiples recursos del mismo tipo, como múltiples cubos de Amazon S3, para distinguir entre ellos.

Este enfoque mantiene sus fuentes de datos ocultos a sus usuarios y asegura que todas las interacciones con la fuente de datos sean manejadas de forma segura. ILLA evita hacer conexiones del lado del cliente a la fuente de datos que puedan exponer información sensible y utilizar las mismas condiciones para todas las interacciones. Adicionalmente, los recursos permiten la caché de consultas, lo que puede reducir el número de peticiones realizadas a una fuente de datos y potencialmente reducir los costos.
