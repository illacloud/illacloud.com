## <Heading hidden>Přepínač</Heading>

Rádiové tlačítko je vstupní komponenta, která umožňuje uživatelům vybrat jednu možnost ze skupiny předdefinovaných možností. Vypadá to jako zaokrouhlené čtvercové tlačítko, které mohou být vybrány nebo zrušeny uživateli. Když je vybráno rádiové tlačítko, všechna ostatní tlačítka ve stejné skupině jsou automaticky odvybrána, zajištění toho, aby bylo možné zvolit pouze jednu možnost najednou. Rádiová tlačítka se běžně používají ve formulářích, dotaznících a průzkumech, které pomáhají uživatelům učinit jednu volbu ze seznamu možností.

### Vlastnosti

#### Ruční možnosti

Přidat nové možnosti nebo ručně odstranit možnosti a nakonfigurovat možnosti jeden po druhém. Vlastnosti opcí jsou následující: 

| Vlastnosti | L 343, 22.12.2009, s. 1).               |
| ---------- | --------------------------------------- |
| Popisek    | Text zobrazený pro možnost              |
| Hodnota    | Hodnota spojená s možností              |
| Zakázáno   | Určuje, zda je volba pro výběr zakázána |

#### Mapované možnosti

Můžete také přidat možnosti ze zdroje dat mapováním popisku a vlastností hodnoty do odpovídajících datových polí. Tímto způsobem můžete získat možnosti dynamicky z databáze. Na konci tohoto článku ukážeme, jak jej využít prostřednictvím příkladu.

| Vlastnosti  | L 343, 22.12.2009, s. 1).                                  |
| ----------- | ---------------------------------------------------------- |
| Zdroj údajů | Nastavte zdroj dat možností                                |
| Popisek     | Pomocí {{item}} nastavíte popisek možností                 |
| Hodnota     | Pomocí {{item}} nastavíte hodnotu možností                 |
| Zakázáno    | Pomocí {{item}} nastavíte, zda je volba zakázána pro výběr |

### Manažer událostí

Podporujeme poslechnutí události `onChange` komponenty rádiového tlačítka. 

Když se změní vybraná možnost, může spustit zadanou akci. To znamená, že když uživatel vybere jinou možnost než skupinu rádiových tlačítek, na základě zvolené možnosti lze přijmout konkrétní opatření. Například ve webové podobě, výběr jiné možnosti může spustit zobrazení různých polí formuláře nebo odeslání formuláře do jiného místa určení. Tímto způsobem může volba vybraná uživatelem ovlivnit chování softwaru nebo webové aplikace, kterou používá. Schopnost spustit konkrétní akce založené na vstupu uživatele je důležitým aspektem interaktivního návrhu softwaru a může zlepšit použitelnost a funkčnost softwaru.

### Případ použití

Dále ukážeme, jak používat mapovaný režim k dynamickému získání možností. 

#### Krok 1 Přidat akci

Vytvořili jsme tabulku `selection`, včetně 5 sloupců: `options_en`, `options_jp`, `options_zh`, `options_kr`, `value`. Hodnoty možností jsme uložili v `hodnotě` a uložili popisky možností v různých jazycích v jiných sloupcích. 

Vytvořte akce pro zobrazení všech dat v `selection` s názvem `postgresql1`

<img src='https://cdn.illacloud.com/official-website/img/docs/assemble/radio1.png' alt="" />

```jsx
vybrat * z výběru
```

#### Krok 2 Konfigurace komponenty

1.  Nastavte zdroje dat na `{{postgresql1.data}}` 
2.  Nakonfigurujte štítek pro změnu štítku na základě jazyka. 

    1.  Pro nastavení sloupce použijte {{ item.columnName }}. 
    2.  Pomocí {{ currentUserInfo.language }} nastavíte jazyk uživatelů v ILLA.

    ```jsx
    {{currentUserInfo.language=='ja-JP' ? item.options_jp 
    : currentUserInfo.language == 'ko-KR' ? item.options_kr 
    : currentUserInfo.language == 'zh-CN' ? item.options_zh 
    : item.options_en }}
    ```
