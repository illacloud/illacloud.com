## <Heading hidden>Docker in-1 зображення</Heading>

### Опис

Створіть ілюстрацію через все в одному зображення і запустіть на вашому комп'ютері. Ви можете перевірити файл скриптів у теці [scripts](https://github.com/illacloud/deploy-illa-manually/blob/main/docker/scripts) для деталей.

Нотатка:

Наполегливо рекомендуємо розгорнути наші інструменти авторозгортання[illa-cli](https://github.com/illacloud/illa).

І на даний момент ми не підтримуємо арх Apple Silicon M1 (darwin-6).

### Запустити офіційне зображення

Встановити GNU значення і тип:

```jsx
зробити run-офіційне зображення
```

або просто виконати:

```jsx
/bin/bash scripts/run-official-image.sh
```

ця команда буде тягнути ілюстраційний образ і запустити його в середовищі докера.

Для входу за замовчуванням ім'я користувача **`root`** і пароль **`паролю`**.

### Створити зображення вручну і запустити

Встановити GNU (або виконати скрипти оболонки в теку скриптів вручну).

Для побудови цілого мультиплікатора в одному зображенні та запуску, типу:

```jsx
зробити все
```

для подмена, друкувати:

```jsx
створити виконуваний подмен
```

або виконувати скрипти `run-by-podman.sh` у теці сценаріїв.

### Зупинити і видалити контейнер

```jsx
створити тимчасовий і віддалений контейнер
```

### Очистити файл бази даних Postgres

** ЦЕ КОМАНДА ВІД ВИДАЛИТИ ДИТИНУ ФАЙЛ НА ВАШУ СВІЙ!!! \*\*\*** РЕЗЕРВУВАТИСЯ І ВІДКРИТИВНІЙ \*\*\*

За замовчуванням, база даних Postgres зберігатиме в теці `/var/lib/illa/database/postgresql/`.

Запустіть наступну команду, щоб видалити її.

```jsx
створити файл з чистою поштовою базою даних
```

### Змінні середовища Config Runtime

Редагувати `scripts/run.sh` при необхідності:

```jsx
# postgres пароль бази даних
PG_PASS=mysecretpassword

# repo home dir
ILLA_HOME_DIR=/var/lib/illa

# postgres папка даних
PG_VOLUMN=${ILLA_HOME_DIR}/database/postgresql
DRIVE_VOLUMN=${ILLA_HOME_DIR}
```
