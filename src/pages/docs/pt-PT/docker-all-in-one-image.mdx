## <Heading hidden>Docker all-in-one Image</Heading>

### Descrição:

Construa a ilha serve uma imagem completa e roda-a em sua máquina. Você pode conferir os arquivos de scripts na pasta [scripts](https://github.com/illacloud/deploy-illa-manually/blob/main/docker/scripts) para mais detalhes.

Nota:

Nós recomendamos fortemente implantar com nossas ferramentas de auto-deploy[illa-cli](https://github.com/illacloud/illa).

E, por enquanto, não suportamos o Apple Silicon M1 (arca-darwin-arm6).

### Execute uma imagem oficial

Instalar GNU fazer e tipo:

```jsx
tornar imagem-rodo-oficial
```

ou apenas execute:

```jsx
/bin/bash scripts/run-official-image.sh
```

Este comando irá puxar illasoft imagem oficial e executá-lo em seu ambiente docker.

E faça login com o nome de usuário padrão **`root`** e senha **`password`**.

### Construir Imagem Manualmente e Executar

Instale o GNU fazer (ou execute scripts shell na pasta de scripts manualmente).

Para construir illa toda em uma imagem e executar, digite:

```jsx
fazer tudo
```

para podman, digite:

```jsx
fazer run-by-podman
```

ou execute scripts do shell 'run-by-podman.sh' na pasta de scripts.

### Parar e remover contêiner

```jsx
fazer stop-and-remove-contêiner
```

### Limpar Arquivo de Dados Postgres

** ESTE COMANDO EXCLUIR A DATÁVEL FILHA NO SEU DISC!!! \*\*\*** BACKUP, E FAÇA UM \*\*\*

Por padrão, o banco de dados Postgres irá armazenar na pasta `/var/lib/illa/database/postgresql/`.

Execute o seguinte comando para excluí-lo.

```jsx
fazer clean-postgres-database-file
```

### Configurar Variáveis de Ambiente de Execução

Editar `scripts/run.sh` quando necessário:

```jsx
# senha do banco de dados postgres
PG_PASS=mysecretpassword

# repo diretório home dir
ILLA_HOME_DIR=/var/lib/illa

# postgres pasta de armazenamento do arquivo de banco de dados
PG_VOLUMN=${ILLA_HOME_DIR}/database/postgresql
DRIVE_VOLUMN=${ILLA_HOME_DIR}/drive
```
