## <h2 hidden>ボタンとは？</h2>

Button コンポーネントにより、ユーザーはボタンをクリックしてアクションをトリガーしたり、関数を実行したりできます。 

### プロパティー

| プロパティー    | 説明                                                                            |
| --------- | ----------------------------------------------------------------------------- |
| テキスト      | ボタンに表示されるテキスト                                                                 |
| イベントハンドラー | トリガクエリ、コンポーネントのイベントに応じてコンポーネントを制御したり、他のAPIを呼び出したりします。                         |
| 読み込み中     | コンポーネントがロードインジケータを表示するかどうか。                                                   |
| 無効        | コンポーネントが無効化されているかどうかのステータスを制御します。 無効になっている場合、コンポーネントを変更したりフォーカスしたりすることはできません。 |
| ツールチップ    | ユーザーはコンポーネントツールチップを入力できます。 ツールチップがフォーカスされると表示されます。 Markdown 形式がサポートされています。    |
| Hidden    | コンポーネントが非表示になるかどうかを動的に制御します。 動的ブール値を使用して非表示状態を変更できます。                         |
| バリアント型    | ユーザーがボタンに2つの異なるスタイル(塗りつぶしまたはアウトライン)を選択できるようにします。                              |
| テーマの色     | ボタンの背景色と不透明度を指定できます                                                           |

### イベントハンドラ：

| イベント  | 説明                                               |
| ----- | ------------------------------------------------ |
| Click | ユーザーがこのボタンをクリックすると、ユーザーによってカスタマイズされたアクションを実行します。 |

### データ

コンポーネントには一般的に使用されているデータがいくつか含まれており、アプリで **`{{componentName.propertyName}}`** を介して呼び出すことができます。

| プロパティ名        | 説明                             |
| ------------- | ------------------------------ |
| colorScheme   | ボタンの背景色                        |
| 無効            | ブール値は無効状態を示します                 |
| displayName   | このコンポーネントの名前 (すなわちボタン1)        |
| hidden        | ブール値はボタンの非表示状態を示します            |
| hiddenDynamic | ブール値はボタンの非表示動的状態を示します          |
| 読み込み中         | 読み込み値                          |
| テキスト          | ボタンに表示されるテキスト値                 |
| tooltipText   | ツールチップテキストの値                   |
| variant       | ボタンのバリアントスタイル (塗りつぶしまたはアウトライン) |

### ケース

次に、**`form`** コンポーネントを動的に制御するための button の使い方を示します。 

#### ステップ 1 アクションを追加

5つの列を含む**`countries`**という名前のテーブルを作成しました: **`num_code`**, **`alpha_2_code`**, **`alpha_3_code`**, **`en_short_name`**, **`nationality`**. 国名は、**`en_short_name`** カラムを保存したオプション値として使用します。

次に、アクションリストから Supabase の新しいアクションを作成し、**`countries`** にすべてのデータをリストし、supabasedb1 という名前を付けます。 

```sql
個別の en_short_name を「Countries」の値として値で選択します。
```

これが私たちの出力です。

<img src='https://cdn.illacloud.com/official-website/img/docs/countries.png' alt='' />

**`Save`** と **`Run`**をクリックしてアクションを有効にします。

#### ステップ2 コンポーネントを追加

最初に**`フォーム`**コンポーネントを追加し、「Survey」とラベル付けします。 次に、Input コンポーネント、select コンポーネント、数値入力コンポーネントをフォームにドラッグし、**`Name`、`Nationality`、** および **`Age`.** とラベル付けします。

次に、**`countries`**の値をマッピングしてコンポーネントを選択します。 

1.  マップモードでは、データソースを`{{supabasedb1.data}}`に設定します。 
2.  マップされたオプションで、値を`{{item.value}}`に設定します

<img src='https://cdn.illacloud.com/official-website/img/docs/button1.png' alt='' />

次に**`フォーム`**コンポーネントの**`送信`**メソッドを制御するために**`送信`**ボタンを設定します。 

1.  Edit event handlerで、 **`Control component`** を実行中に選択し、更新する**`form`** コンポーネントをイベントのターゲットとして選択します。
2.  **`Submit`** アクションを選択します。
