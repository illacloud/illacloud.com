## <h2 hidden>時間範囲とは何ですか？</h2>

時間範囲コンポーネントは、ユーザーが24時間インターフェイスから時間の範囲を選択することができるUIコンポーネントです。 通常、ユーザーが開始時間と終了時間の範囲を指定する必要があるフォームまたはデータ入力インターフェイスで使用されます。

format@@0コンポーネントがキャンバスに追加されると、2つの24時間インターフェースが並行して表示されます。 1つのインターフェイスを使用して開始時刻を選択し、もう1つは終了時刻を選択します。 ユーザーは、マウスカーソルを移動するか、上下矢印を使用して時間を調整できます。

### Properties

| ステップサイズ     | 分単位のステップサイズ                                                                   |
| ラベル         | ユーザーに表示されるフィールドの名前                                                            |

| Hidden      | コンポーネントが非表示になるかどうかを動的に制御します。 動的ブール値を使用して非表示状態を変更できます。                         |
| テーマの色       | ボタンの背景色と不透明度を指定できます                                                           |

### 大文字小文字を区別する

次に、データソースからのデータを時間範囲にマッピングし、ボタンで終了値を現在時刻に設定する方法を説明します。 

#### ステップ 1 アクションを追加

**`promoteCode`と呼ばれるREST APIで、6つの列を含むテーブルを作成しましょう: **`code`**、**`id`**、**`createdAt`**、**`expiredAt`**、**`startedAt`、\`updatedAt\*\*。 startedAt と expiredAt を使用してサンプルを使用します。 

次に、アクションリストからREST APIの新しいアクションを作成し、promotecodeのすべてのデータをリストし、restapi1という名前を付けます。 

**`Save`** と **`Run`**をクリックしてアクションを有効にします。

#### ステップ2 コンポーネントを追加

次に、**`time range`** コンポーネントと **`button`** コンポーネントをキャンバスに追加します。 日付範囲コンポーネントのデフォルトの開始時刻値には、残りの api データから最初のプロモーションコードの startedAt 時間を使用します。

```sql
{{restapi1.data.promoteCodes[0].startedAt}}
```

同様に、デフォルトの終了時刻値では、expiAt は残りの api データから最初のプロモーションコードを使用します。

```sql
{{restapi1.data.promoteCodes[0].expiredAt}}
```

次に示すように、ボタンを"Now"とラベル付けしました。

#### ステップ 3 コンポーネントの設定

**`button`** コンポーネントについては、終了日時の値を **`time range`** コンポーネントに今日の日付に設定します。

1.  Edit event handlerで**`Control component`** を実行中に選択します。 イベントのターゲットとして更新する **`time range`** コンポーネントを選択します。
2.  **`setEndTime`**アクションを選択し、**`終了時間`** を{{currentUserInfo.updatedAt}}\`に設定します。 

![time_range_config](https://cdn.illacloud.com/official-website/img/docs/time_range_config.png)

#### ステップ4 テスト

「今すぐ」ボタンをクリックすると、終了時刻は今日の日付に変更する必要があります。 ここについては、20:03:05がありますが、テストしているときは違うかもしれません。
