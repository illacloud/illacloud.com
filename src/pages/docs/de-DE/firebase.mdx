## <h2 hidden>Was ist Firebase?</h2>

Firebase ist eine umfassende Plattform von Google, die eine breite Palette von Tools und Dienstleistungen für die Entwicklung und Verwaltung von Web-und mobilen Anwendungen bietet. Es stellt Entwicklern eine skalierbare und sichere Backend-Infrastruktur zur Verfügung, zusammen mit einer Reihe vorkompilierter Funktionen und Funktionalitäten, um den Entwicklungsprozess zu beschleunigen. Mit der Integration von Firebase auf ILLA Cloud können Entwickler ihre Anwendungen mit zuverlässigen und skalierbaren Speicherlösungen erweitern. Damit sie sich auf innovative und datengestützte Erfahrungen konzentrieren können.

### Firebase-API erstellen

Es gibt zwei Möglichkeiten, nach der Anmeldung in Ihrem Illa-Konto eine Ressource in Illa zu erstellen.

-   In Ressourcen erstellen

Melde dich mit deinem Illa-Konto an, wähle `**Ressourcen**` oben auf der Seite und klicke auf `**Neues erstellen**` Button.

![external_resource](https://cdn.illacloud.com/official-website/img/docs/external_resource.png)

Wähle **`Firebase`** aus der API-Liste.

![action_list](https://cdn.illacloud.com/official-website/img/docs/action_list.png)

Verbinden Sie sich mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

Nach der Erstellung einer Ressource wird die fertige Firebase wie gezeigt angezeigt.

![firebase_ex_ready](https://cdn.illacloud.com/official-website/img/official-site/intergration/firebase_ex_ready.jpg)

-   Im Builder erstellen

Melde dich bei deinem Illa Account an, erstelle ein Projekt in Illa Builder auf der `**App**` Seite und navigieren Sie zur **`Action List`** am unteren Ende der Seite. Klicke `**new**`, dann wähle **`Firebase`** aus der Datenbankliste. Dann verbinden Sie sich mit der Datenbank mit den erforderlichen Parametern, die in `Verbindungseinstellungen` unten beschrieben sind.

Klicke auf `**Verbindung testen**` um zu sehen, ob wir erfolgreich mit der Datenbank verbinden können. Wenn ja, klicke auf **`Ressourcen speichern`**, ansonsten überprüfe doppelt, ob der Hostname, der Port, der Benutzername und das Passwort korrekt sind.

### Verbindungseinstellungen

Hier müssen wir Informationen zur Verbindung mit Firebase-API bereitstellen.

![firebase_config](https://cdn.illacloud.com/official-website/img/official-site/intergration/firebase_config.jpg)

| Eigenschaften                | Benötigt | Beschreibung                                                                                                           |
| ---------------------------- | -------- | ---------------------------------------------------------------------------------------------------------------------- |
| Name                         | benötigt | Der Name für die Ressource beim Erstellen von Aktionen in der ILLA.                                                    |
| Firebase-Datenbank-URL       | optional | der Endpunkt oder die URL, auf der die Firebase Realtime Datenbank gehostet wird                                       |
| Firestore Projekt ID         | optional | ein eindeutiger Identifikator, der jedem Firebase-Projekt zugewiesen ist, das den Firestore-Datenbank-Dienst nutzt     |
| Privater Schlüssel           | benötigt | eine Sicherheits-Anmeldeinformation, die für die Authentifizierung und den Zugriff auf Firebase-Dienste verwendet wird |

### Aktionen erstellen

Wir können die Aktion hinzufügen, indem wir Firebase aus der Aktionsliste auswählen und die **`Aktion erstellen`** Schaltfläche auswählen.

![firebase_resource_list](https://cdn.illacloud.com/official-website/img/official-site/intergration/firebase_resource_list.jpg)

Jetzt haben wir den Firebase-Server als Aktion zu unserer Bauseite hinzugefügt.

![firebase](https://cdn.illacloud.com/official-website/img/official-site/intergration/firebase.jpg)

#### Benutzer per E-Mail erhalten

abrufen, indem Sie die E-Mail-Adresse des Benutzers angeben. Es erlaubt Ihnen, Benutzerdetails ähnlich der UID-basierten Methode abzurufen, aber die E-Mail als Identifikator zu verwenden.

** Input **

| Eigenschaften | Benötigt | Beschreibung                                                                      |
| ------------- | -------- | --------------------------------------------------------------------------------- |
| email         | benötigt | die registrierte E-Mail, die dem Benutzer zugeordnet ist, den Sie abrufen möchten |

#### Nutzer über Telefonnummer erhalten

abrufen von Benutzerinformationen durch Angabe der Telefonnummer des Benutzers. Es erlaubt Ihnen, Benutzerdetails ähnlich der UID-basierten Methode zu holen, aber mit der Telefonnummer als Identifikator.

** Input **

| Eigenschaften   | Benötigt | Beschreibung                                                                                |
| --------------- | -------- | ------------------------------------------------------------------------------------------- |
| Telefonnummer   | benötigt | die registrierte Telefonnummer, die mit dem Benutzer verknüpft ist, den Sie abrufen möchten |

#### Benutzer auflisten

| Länge der Liste          | optional | Anzahl der Benutzer, die Sie in einer einzigen Anfrage abrufen möchten                                |
| Token der nächsten Seite | optional | \-Seitenanzeige wenn es mehr Benutzer gibt, die in einer einzigen Antwort zurückgegeben werden können |

| wert           | benötigt | die tatsächlichen Daten, die Sie in das Dokument einfügen möchten                                                                                         |
