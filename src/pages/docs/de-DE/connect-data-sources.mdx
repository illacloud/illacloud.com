## <Heading hidden>Verbinden Sie sich mit Ihren Datenquellen (Aktion)</Heading>

Erfahren Sie, wie Sie sich mit Ihrer Datenquelle verbinden und rufen Sie diese im ILLA Builder auf.

Mit ILLA können Sie eine Vielzahl von Datenquellen, einschließlich Datenbanken und APIs, zur Nutzung als Ressourcen in Ihr Unternehmen integrieren und integrieren.

### Datenquellen

Datenquellen sind Orte, an denen Daten gespeichert und verarbeitet werden. Beispiele für Datenquellen sind PostgreSQL-Datenbanken, Twilio REST-APIs und Amazon S3-Buckets. In ILLA können Sie mehrere Datenquellen hinzufügen und für Benutzer zugänglich machen.

#### Integrationen

ILLA bietet integrierte Integrationen für viele beliebte Datenquellen, die eine minimale Konfiguration erfordern, wie SQL-Datenbanken und Stripe. Diese Integrationen, auch als Ressourcen bekannt, ermöglichen es Ihnen, sich mit den Datenquellen über ILLA zu verbinden und abzufragen. Wenn eine bestimmte Art von Datenquelle nicht von einer integrierten Integration unterstützt wird können Sie mit den REST, GraphQL oder SOAP Integrationen eine Verbindung zu fast jeder API herstellen. So können Sie auf eine Vielzahl von Datenquellen aus dem ILLA-Builder zugreifen.

### Ressourcen

Eine Ressource in ILLA ist eine gespeicherte Konfiguration, die festlegt, wie sich ILLA mit einer Datenquelle verbindet. Um eine Datenquelle in der ILLA zu verwenden, müssen Sie eine Ressource dafür erstellen. Sie können beim Schreiben einer Abfrage die entsprechende Ressource für den Zugriff auf die Daten auswählen.

Jede Ressource in ILLA hat eigene Konfigurationsoptionen, die bestimmen, wie sich ILLA mit der entsprechenden Datenquelle verbindet. Diese Optionen können Authentifizierungsdaten und -ströme, Datenbankverbindungseinstellungen und API-Einstellungen, wie Basis-URL und Standard-Header, enthalten.

#### Wie Ressourcen funktionieren

ILLA Ressourcen speichern die notwendigen Einstellungen für eine sichere Verbindung zu Ihrer Datenquelle. So können Sie Daten leicht abfragen, ohne Ihre Datenquelle direkt Ihren Benutzern zu entlasten. Wenn eine Abfrage für eine Ressource ausgeführt wird, ILLA verwendet die Konfigurationseinstellungen der Ressource, um die Anfrage an die Datenquelle auf der Serverseite zu proxy zu machen. Dadurch wird sichergestellt, dass nur die ILLA direkten Zugriff auf die Datenquelle hat.

ILLA vereinfacht den Verbindungsprozess zu einer Datenquelle für Ihre Benutzer. Sie müssen nur die relevante Ressource auswählen, wenn sie eine Abfrage schreiben, ohne sich um die Details der Datenquelle zu kümmern. Sie können auch mehrere Ressourcen desselben Typs, wie z. B. mehrere Amazon S3-Gruppen hinzufügen, um zwischen ihnen zu unterscheiden.

Dieser Ansatz hält Ihre Datenquellen vor Ihren Nutzern verborgen und sorgt dafür, dass alle Interaktionen mit der Datenquelle sicher behandelt werden. ILLA vermeidet die Anbindung an die Datenquelle, die sensible Informationen enthüllt und für alle Interaktionen die gleichen Bedingungen verwendet. Zusätzlich ermöglichen Ressourcen den Abfrage-Caching, was die Anzahl der Anfragen an eine Datenquelle reduzieren und möglicherweise geringere Kosten verursachen kann.
