Apprenez-en plus sur la connexion à votre source de données et appelez-la dans votre constructeur ILLA.

ILLA vous permet de connecter et d'intégrer une large gamme de sources de données, y compris les bases de données et les API, à votre organisation pour qu'elle puisse être utilisée comme ressources.

## Sources de données

Les sources de données sont des lieux où les données sont stockées et traitées. Des exemples de sources de données incluent les bases de données PostgreSQL, les API Twilio REST et les compartiments Amazon S3. En ILLA, vous pouvez ajouter plusieurs sources de données et les rendre accessibles aux utilisateurs pour des requêtes.

### **Intégrations**

ILLA fournit des intégrations intégrées pour de nombreuses sources de données populaires qui nécessitent une configuration minimale, comme les bases de données SQL, et Stripe. Ces intégrations, aussi connues sous le nom de ressources, vous permettent de vous connecter et de interroger vos sources de données à l'aide de Retool. Si un type particulier de source de données n'est pas pris en charge par une intégration intégrée, vous pouvez utiliser les intégrations REST, GraphQL, ou SOAP pour vous connecter à presque n'importe quelle API. Cela vous permet d'accéder à un large éventail de sources de données depuis le constructeur ILLA.

## Ressource

Une ressource dans ILLA est une configuration sauvegardée qui spécifie comment ILLA se connecte à une source de données. Pour utiliser une source de données dans ILLA, vous devez créer une ressource pour celle-ci. Vous pouvez sélectionner la ressource appropriée pour accéder aux données lors de l'écriture d'une requête.

Chaque ressource dans ILLA a ses propres options de configuration qui déterminent la façon dont ILLA se connecte à la source de données correspondante. Ces options peuvent inclure les identifiants et le flux d'authentification, les paramètres de connexion à la base de données et les paramètres de l'API, tels que l'URL de base et les en-têtes par défaut.

### **Comment fonctionnent les ressources**

Les ressources ILLA stockent les paramètres nécessaires pour vous connecter à votre source de données en toute sécurité. Cela vous permet de demander facilement des données sans exposer votre source de données directement à vos utilisateurs. Quand une requête est exécutée pour une ressource, Ré-outil utilise les paramètres de configuration de la ressource pour proxy la requête à la source de données du côté serveur. Cela garantit que seul Retool a un accès direct à la source de données.

ILLA simplifie le processus de connexion à une source de données pour vos utilisateurs. Ils n'ont besoin que de sélectionner la ressource pertinente lors de l'écriture d'une requête sans se soucier des détails de la source de données. Vous pouvez également ajouter plusieurs ressources du même type, comme plusieurs compartiments S3 d'Amazon, pour les distinguer.

Cette approche garde vos sources de données cachées à vos utilisateurs et veille à ce que toutes les interactions avec la source de données soient gérées en toute sécurité. ILLA évite de faire des connexions côté client à la source de données qui pourraient exposer des informations sensibles et utilise les mêmes conditions pour toutes les interactions. De plus, les ressources permettent la mise en cache des requêtes, ce qui peut réduire le nombre de requêtes faites à une source de données et potentiellement réduire les coûts.
